# 1-团队员工说明（人类用户阅读版）

> 用途：帮助你快速识别每位员工、理解其工作边界，并检查项目负责人是否按正确顺序调用团队。
>
> 重要：这些员工不是后台自动运行的程序。项目负责人必须在关键节点**显式调用**并把结论归档到项目文件中。

> 编号说明：配置文件 `0-goal-alignment-monitor.md` 的 `0-` 表示“最先调用”；本文件为方便人类阅读，员工名单从 1 开始编号。

## 最高优先级调用逻辑

每位新负责人接手时，先阅读 `AGENTS.md` 和 `.codex/agents/1-project-lead.md`，然后严格执行：

1. **GoalAlignmentMonitor（总目标对齐监控员）**：先判断项目是否仍朝你的最终目标前进，结论为 `ALIGNED`、`DRIFT` 或 `UNKNOWN`。
2. **RequirementCertifier（需求信心审查员）**：再判断本轮具体工作是否具备 100% 证据，结论为 `READY` 或 `NOT READY`。

只有“目标没有偏离”且“本轮需求 READY”时，才允许写代码、改变运行行为或进入新阶段。出现偏离、证据不足或任务中断时，必须先由 DocUpdater 和 Handover 归档，不允许只留在聊天记录里。

## 员工名单与功能

| 序号 | 英文名字（中文名字） | 主要功能 | 关键调用时机 |
|---:|---|---|---|
| 1 | GoalAlignmentMonitor（总目标对齐监控员） | 检查用户目标、产品决定、阶段路线、测试状态、案例边界和归档是否一致；发现目标漂移 | **每次接手第一个调用**；新阶段、重大方向调整前 |
| 2 | ProjectLead（项目负责人） | 管理任务顺序、调度员工、整合证据、控制范围、交付结果 | 全程唯一总入口 |
| 3 | RequirementCertifier（需求信心审查员） | 独立判定本轮任务是否 `READY/NOT READY`；不允许以主观信心代替证据 | **每次接手第二个调用**；写代码/改行为前终审 |
| 4 | KnowledgeExpert（案例知识专家） | 阅读优秀案例 1、2 源码，说明可借鉴内容与禁止复制的边界 | 需求或设计需要案例证据时 |
| 5 | CodeExplorer（代码探索员） | 只读定位项目文件、函数、调用关系和行号 | 需要确认当前代码事实或安全边界时 |
| 6 | UIReferenceAnalyst（UI 参考分析员） | 对照项目截图、参考图、UI 源码和视觉规格 | UI 设计、颜色、布局、窗口规则审查时 |
| 7 | ProjectManager（项目结构管理员） | 管理目录、命名、文件归属、索引和项目结构说明 | 新增/移动文档或阶段成果时 |
| 8 | DocUpdater（项目文档更新员） | 记录已完成事项、测试、风险、未完成项和下一步 | **每轮任务、用户反馈、中断或里程碑后必须调用** |
| 9 | TestEngineer（测试工程师） | 设计自动测试、静态检查和人类用户验收教程 | 实现完成前后；涉及真实输入时 |
| 10 | RealtimeChecker（实现偏差检查员） | 检查实现是否偏离已批准规格、案例方向与安全边界 | 功能/文档初稿完成后、提交前 |
| 11 | ReleaseManager（发布管理员） | 管理 Git 基线、暂存范围、提交、普通推送和远端 SHA 核验 | 发布到 GitHub 前后 |
| 12 | Handover（交接归档员） | 更新 `CURRENT_HANDOVER.md` 并创建递增编号的历史交接 | 阶段结束、任务中断、提交前后 |
| 13 | AntiHallucination（防幻觉审查员） | 检查未验证声明、文件过长、职责混杂、遗漏的引用与风险 | 新文件初稿、提交前、里程碑、交接前 |

## 两类“审查”不要混淆

- GoalAlignmentMonitor 解决：“我们是不是还在做你真正想要的项目？”
- RequirementCertifier 解决：“这一步是否已经有足够证据，可以安全开工？”
- AntiHallucination 解决：“文档和交付是否存在无证据的说法、遗漏或职责混乱？”

三者都需要显式调用；没有任何一个角色可以替代另外两个。

## 必须归档的内容

项目负责人每轮都必须让 DocUpdater 和 Handover 记录：员工结论、已做/未做内容、测试结果、风险、Git 状态、用户待办和下一项唯一任务。记录入口为：

1. `PROJECT_OUTLINE.md`：人类可见的总进度。
2. `REVIEW_LOG.md`：审查结论与证据。
3. `my-automation-tool/docs/handover/CURRENT_HANDOVER.md`：下一位 AI 的稳定入口。
4. `my-automation-tool/docs/handover/<编号>-...md`：不可覆盖的历史交接快照。

即使工作尚未完成、网络阻塞或用户改变方向，也必须先写明当前进度和恢复入口，再结束本轮工作。
