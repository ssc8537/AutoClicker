## 一、项目目标

开发一个键盘自动化序列播放器，用于游戏多角色快速切换与技能释放。

核心逻辑：自定义按键序列脚本 -> 绑定快捷键 -> 按下快捷键 -> 程序在目标窗口按顺序模拟键盘/鼠标按键，实现多角色技能近乎同时释放。

- 用普通鼠标/键盘触发，不依赖罗技 Lua 宏。
- 纯固定序列播放，不需要图像识别、状态判断。
- 脚本可以是 JSON 配置文件（简单）或 Python 代码（灵活）。
- UI 面向小白，全部操作可视化。

终极目标：最低延迟切换角色（10ms 级），让两个甚至三个角色的技能同时在场。

## 九、安全需求（阶段1.0 - 最高优先级）

### 9.1 默认禁用原则
程序启动时，全部全局热键处于禁用状态。用户必须显式按下全局启用键（默认 F12）后，所有脚本热键才生效。

### 9.2 全局启用/禁用键
默认键：F12（toggle）
状态示意：红色热键已禁用 / 绿色热键已启用

### 9.3 进程可视化管理
窗口关闭 = 进程退出。closeEvent 中调用 keyboard.unhook_all()。atexit 兜底注册清理函数。

### 9.4 孤儿进程防护
atexit.register(cleanup) + closeEvent 双重机制。

### 9.5 紧急停止
按 F12 可随时禁用所有脚本热键。

### 9.6 阶段 1 默认 F9 交互（已确认）

F9 默认使用 **按住模式（DOWN）**，作为后续游戏连招的最小验证：

1. 物理按下 F9 时，只启动一次 Hello World 循环，并显示绿色“Hello World 脚本运行中”提示。
2. 持续按住期间，程序逐轮输出 Hello World；操作系统产生的重复按下事件不得重复创建执行任务。
3. 物理松开 F9 时，立即请求停止；已发送字符会先释放，之后不再发送字符或开始下一轮，并显示红色“Hello World 脚本已停止”提示。
4. 任意时刻按 F12 禁用时，必须停止正在运行的 F9 循环；关闭主窗口时同样停止。

该行为以 Quickinput `trigger.cpp` 的 `down` 模式为参考。阶段 2 只有一个宏，只支持 `switch`、`down` 两种模式；`down` 的 `count=0` 在松开时停止，有限次数在松开后完成。字段、次数、速度和停止语义以 `docs/阶段2-最小需求规格.md` 为准。松开启动模式、Unicode 打字、多个脚本与组合键均不属于阶段 2。
