# 项目负责人：快速小阶段流程

负责人在同一上下文完成取证、实现、测试、教程、归档和交接。开始时只读根 `AGENTS.md`、`PROJECT_OUTLINE.md`、当前交接、当前阶段规格/教程和 `git status --short`；不默认阅读旧编号交接或调度员工。

## 开始前

用简短记录确认：用户目标、范围外、可观察行为、安全/回退、自动检查和人工通过标准。证据足够则记为 `READY` 并实现；缺少会实质改变安全、兼容或验收的决定时才问用户。案例事实、视觉或专业风险仍不清楚时，按需请一位专家只读取证，不建立形式化多员工链路。

## 交付

只实现当前小阶段，先运行针对性测试，再运行全量单测、`compileall` 和 `git diff --check`。UI/真实输入变更提供中文 Windows 教程；用户验收通过后再进入下一阶段。默认不提交、不推送；发布需用户明确授权且禁止强推。

## 永久边界

可信本地 Python `run(player)`、F9/F12、F2 占位、唯一活动宏和安全释放继续有效。优秀案例只读，禁止复制源码、资产或品牌。用户工作区文件必须保留。交接保持不超过一屏：当前状态、自动证据、未覆盖风险、唯一下一步和必读路径；历史文件只在查证时读取。
