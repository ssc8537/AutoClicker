# MyAutoPlayer：50% 减负模式

## 最高指令：按需案例专家

`案例专家`是永久保留的唯一子 Agent。负责人遇到无法由当前源码、规格和自动测试解决的技术问题、案例行为或代码设计难题时，必须调用他；他可只读查看“优秀案例 1”“优秀案例 2”的全部源码，并结合本项目全部逻辑给出源码证据、适配建议和未知项。没有这类难题时，负责人自行完成工作，不调用任何子 Agent。案例专家不复制案例源码/资源、不直接改代码、不扩大产品范围。

默认只读本文件、`my-automation-tool/docs/handover/CURRENT_HANDOVER.md`，并运行 `git status --short`。只有开始新功能时才读取该功能的一页规格和验收教程；旧员工文件、编号交接、审查日志和旧路线已从工作区删除，需要追溯时只从 Git 标签读取。

负责人单上下文完成：明确一个小目标和范围外 → 简短 READY 判断 → 实现 → 自动测试/编译/差异检查 → 中文 Windows 教程 → 用户验收 → 更新当前交接。无需默认调度员工、重复审查或复制历史文字。

保留的底线：Python `run(player)`、F9/F12、F2 占位、停止释放、真实输入先自动测试后人工验收、案例只读、用户文件不动、默认不发布且禁止强推。只有无法确认的案例/UI/安全事实才按需请一名专家。

这比上一版默认四份文档加多员工流程减少超过 50% 的阅读和等待；平台实际模型以运行环境为准。
