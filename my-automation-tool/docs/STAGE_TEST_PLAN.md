## 阶段 2 测试：序列播放器

### 测试目标
程序能按**自定义延迟**的顺序打出 H-e-l-l-o-空格-W-o-r-l-d。

### 测试方法
1. 启动程序，按 F12 启用
2. 按下 F9
3. 程序按以下顺序打字：
   - H → 等 100ms → e → 等 50ms → l → 等 50ms → l → 等 50ms → o → 等 200ms → 空格
   - W → 等 100ms → o → 等 50ms → r → 等 50ms → l → 等 50ms → d

### 预期结果
在记事本中看到：**Hello World**（每个字母之间有可感知的时间间隔）

### 为什么这个测试有用
游戏中切换角色、放技能也是类似的"顺序按键 + 延迟控制"。你能控制好每个字母的延迟，就能控制好每个技能的延迟。**这就是角色的自动出招。**

### 测试清单

| # | 操作 | 预期结果 | 打勾 |
|---|------|---------|------|
| 1 | 启动程序，按 F12 启用 | 顶部变绿色 | [ ] |
| 2 | 光标放记事本，按 F9 | 打出 H(100ms)e(50ms)l(50ms)l(50ms)o(200ms) 空格 W(100ms)o(50ms)r(50ms)l(50ms)d | [ ] |
| 3 | 在序列执行中途按 F12 禁用 | 序列立即停止，不再继续输出 | [ ] |

## 测试的核心逻辑

为什么用 Hello World 做测试就够了？

因为 **"打出 Hello World" 等价于 "执行角色的自动出招"**：

| Hello World | 等价于游戏中 |
|-------------|-------------|
| 字母 H | 按 1 号角色键 |
| 延迟 100ms | 角色切换延迟 |
| 字母 e | 按 E 放战技 |
| 延迟 50ms | 技能间等待 |
| 空格 | 跳跃 |
| 字母 W | 按 2 号角色键 |
| ... | ... |

如果你能通过程序的控制，让 H 和 W 在不同的延迟下正确打出，就相当于你让角色 1 和角色 2 在不同的延迟下正确释放技能。**这就是多角色同时在场。**
