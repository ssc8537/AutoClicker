## 核心思想

用"正确打出 Hello World"来测试每个阶段的功能是否实现。
每个阶段测试通过 = 软件该阶段功能正确。

## 阶段 3 测试：热键绑定 UI + 脚本编辑器

### 测试目标
能在图形界面中管理脚本。左边显示脚本列表，右边显示详情面板。支持 KeyEdit 热键捕获、编辑、执行次数设置。

### 测试清单

| # | 操作 | 预期结果 | 打勾 |
|---|------|---------|:----:|
| 1 | 启动程序，看到 Quickinput 风格的界面 | 左边脚本列表，右边详情面板 | [ ] |
| 2 | 列表中看到已有 JSON 脚本 | 脚本名显示在列表中 | [ ] |
| 3 | 点击 KeyEdit 控件，按 F9 | 控件正确显示 F9 | [ ] |
| 4 | 修改触发模式为 SWITCH | 模式切换生效 | [ ] |
| 5 | 设置执行次数为 3 次 | 按 F9 后 Hello World 重复打出 3 次 | [ ] |
| 6 | 设置速度倍率为 0.5x | 打字速度变慢一半 | [ ] |
| 7 | 新建一个脚本，命名为"测试连招" | 脚本出现在列表中 | [ ] |
| 8 | 编辑脚本内容，保存后按 F9 | 修改后的脚本正确执行 | [ ] |
| 9 | 删除一个脚本 | 脚本从列表移除 | [ ] |
| 10 | 禁用单个脚本（state 切换） | 该脚本不触发，其他脚本正常 | [ ] |
