# Stage 3M 左右鼠标 API：Windows 小白验收教程

## 启动前准备

1. 打开记事本，创建一个空白文本文件；不要打开游戏。
2. 用 PowerShell 启动：

```powershell
cd C:\Users\s\Desktop\1-test\1-自动连点器\my-automation-tool
python main.py
```

3. 在宏库新建或编辑一个可信 Python 宏，保存以下 `run(player)` 内容；再到“触发”页启用它。

```python
def run(player):
    player.mouse_down("left")
    player.sleep(400)
    player.mouse_up("left")
    player.mouse_click("right")
    player.mouse_repeat(3, "left", interval_ms=100)
```

## 操作与预期

1. 按 F12 使状态变为已启用，把鼠标移到记事本空白区域，按一次 F9。预期：约 400ms 后左键松开，右键会打开一次记事本菜单，左键有限点击后停止；不会持续点击。
2. 关闭右键菜单，修改宏为 `mouse_down("left")`、`sleep(5000)`、`mouse_up("left")`，保存并保持启用。鼠标放到记事本空白区域后按 F9，立即按 F12。预期：宏停止，鼠标之后可以正常单击、拖动和松开，不会像左键一直被按住。
3. 再按 F12 重新启用，按 F9 运行第 1 步宏；在 400ms 按住等待期间直接关闭程序窗口。预期：窗口关闭后鼠标恢复正常，不残留按住状态。
4. 重新启动程序，确认 F2 仍只是界面占位；确认 F9 只运行触发页唯一已启用宏，F12 仍能停止。

## 通过标准

四步均符合预期，且没有持续点击、卡住左/右键、额外热键或游戏相关行为。

本教程不测试游戏兼容性、坐标/滚轮/中键/侧键、录制、窗口绑定、图像或游戏状态判断、多宏并行，也不人工计数 100 次或测量 10ms；这些边界由自动测试覆盖。
