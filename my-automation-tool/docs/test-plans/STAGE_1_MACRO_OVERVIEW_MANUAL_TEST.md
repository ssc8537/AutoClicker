# Stage 1：宏页选脚本、触发页控制启停 Windows 小白验收教程

> 本教程只验收 Candy 粉红宏页、全量脚本触发页和既有 F9/F12 安全回归。不要点击灰色的“编辑、保存、重新加载、删除、保存触发设置”按钮，也不要测试 F2。

## 启动前准备

1. 打开 PowerShell，进入 `my-automation-tool` 文件夹。
2. 运行：`python main.py`。
3. 程序打开后先不要按 F12；全局热键应保持禁用。
4. 准备一个空白记事本窗口；测试 F9 时将光标放在其中。

## 操作与预期

| 步骤 | 你要做什么 | 正确结果 |
|---|---|---|
| 1 | 停留在默认打开的“宏库”页，选中任一脚本。 | 保留 Candy 粉红双栏；左侧是宽白色脚本列表，右侧名称框显示所选脚本。“编辑、保存、重新加载、删除”均为灰色占位。选中脚本本身不会启用或禁用它。 |
| 2 | 查看 `hello_world.py`、`goodbye.py` 和 `invalid_missing_run.py`。 | 前两个为普通文字；无效脚本只有名称为红色，不显示错误内容。 |
| 3 | 切换到“触发”页，点击任一表格单元格，并在后续步骤按 F9 后回看表格。 | 左侧表格同时显示上述全部脚本，不是只显示一个活动脚本；任何点击、选中或 F9 触发后，表头和数据行都固定显示“名称、按键、模式、状态”四列，“名称”列不得被横向挤出。 |
| 4 | 查看三条脚本行。 | 有效脚本显示固定 `F9`、各自“切换/按住”模式和“禁用”；无效脚本名称红色，状态为“不可启用”。 |
| 5 | 点击一个有效脚本的“状态”单元格。 | 该行状态变为“启用”，右侧只读栏显示它的 F9、模式、次数、速度和状态。 |
| 6 | 点击另一个有效脚本的“状态”单元格，再点击当前启用行。 | 第二行启用时第一行自动变为“禁用”；再次点击当前启用行后它变为“禁用”。同一时刻只能启用一个有效脚本。点击无效行没有变化。 |
| 7 | 宏页选中另一个脚本，但不点击触发页状态。 | 触发页原来的启用/禁用状态不变；宏页选择不会改变可触发脚本。 |
| 8 | 回到记事本，先确认触发页没有启用有效脚本；按 F12 启用后轻按 F9，再按 F12 禁用。 | 记事本没有输入任何文字；禁用后仍没有输入。 |
| 9 | 在触发页启用 `hello_world.py`，回到记事本，按 F12 启用后按住 F9，随后松开并按 F12 禁用。 | 按住期间只出现该宏的 `hello world`；松开后停止；最终按 F12 后再次按 F9 不再输入。 |

## 反馈格式

全部符合时回复：`Stage 1 宏页选脚本、触发页全量展示和 F9/F12 回归通过`。

有任一步不符合时回复：`Stage 1 第 X 步失败；宏页选择后状态：…；触发页显示了几行及各行状态：…；是否已按 F12 启用：是/否；记事本文字：…`。方便时附一张 MyAutoPlayer 截图。

## 未覆盖风险

离屏自动测试不能代替你的 Windows 中文字体、窗口缩放和真实键盘输入确认；本轮不会测试或启用编辑、保存、重新加载、删除、F2、录制、鼠标、窗口、多宏并发或导入导出功能。
