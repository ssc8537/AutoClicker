# Stage 2B：Python 编辑体验 Windows 小白验收教程

> Stage 2B 的原有步骤已由用户通过。本次只验收 Enter 自动缩进反馈修复；不要测试 AI 提示词、鼠标、游戏键位、删除、导入导出或新快捷键。

## 启动前准备

1. 打开 PowerShell，进入 `my-automation-tool` 文件夹。
2. 运行 `python main.py`；程序启动后先不要按 F12。
3. 选择一个未在“触发”页启用的有效 Python 宏；若没有，可先按 Stage 2A 教程新建一个。

## 操作与预期

| 步骤 | 操作 | 正确结果 |
|---|---|---|
| 1 | 打开“宏库”。 | 左侧“Python 宏”只显示名称，不显示 `.py`；无效文件仍是红字。单击一行只选中，不会自动弹出编辑器。 |
| 2 | 在右侧名称框输入一个新名称，按 Enter。 | 列表与“触发”页立即显示新名称；旧文件名不再出现。名称框的 Enter 与点击“保存”结果相同。 |
| 3 | 选中该宏并点击“编辑”。 | 弹窗标题、名称标签和两个按钮均为中文；底部按钮是“保存”“取消”。 |
| 4 | 在 `def run(player):` 的行尾按 Enter；再在一条已缩进的 `player.sleep(50)` 行尾按 Enter。 | 函数头后的新行自动有 4 个空格；普通已缩进行的新行保持相同缩进，不执行或保存代码。 |
| 5 | 点击“取消”，重新打开确认代码仍未保存。 | 取消不修改文件；本次没有新增 AI 提示词、鼠标或游戏键位功能。 |

## 反馈格式

通过时回复：`Stage 2B-R Enter 自动缩进通过`。

失败时回复：`Stage 2B-R 第 X 步失败；Enter 前代码：…；Enter 后内容：…`，方便时附截图。

## 未覆盖风险

本阶段不验证游戏键位、鼠标、删除、导入导出、多宏、AI 提示词或游戏兼容性。离屏自动检查不能替代 Windows 字体、输入法和实际颜色观感。
