# C1 宏库自动检测：Windows 小白验收教程

## 启动与准备

1. 打开 PowerShell，进入 `my-automation-tool` 目录，运行 `python main.py`。
2. 打开 Windows 记事本，点击正文，让光标留在空白处；鼠标不要停在 MyAutoPlayer 窗口内。
3. 程序启动后默认没有活动宏。宏库页没有“刷新”或“设为活动宏”按钮；文件保存后会自动更新。

## 逐步验收

1. 确认表格显示三个文件：`hello_world.py`、`goodbye.py`、`invalid_missing_run.py`。前两个“文件校验”为“有效”，第三个显示“无效”，且活动状态为“不可启用”；右侧状态区应汇总无效文件名和中文原因。触发页初始显示“未选择有效宏 / 不执行”。
2. 不点击任何“活动状态”。按 F12 启用后轻按 F9：记事本不得出现文字；按 F12 再次禁用。
3. 点击 `hello_world.py` 所在行的“活动状态”单元格，使其显示“已启用”。按 F12 启用，按住 F9：记事本应只输入 `hello world`；松开 F9 后停止。再按一次该状态单元格，应停用它；此时 F9 不得输入。
4. 依次点击 `hello_world.py`、`goodbye.py` 的活动状态。确认同一时刻只有一个“已启用”，触发页同步显示当前脚本、F9、模式、次数和速度。保持 `goodbye.py` 启用后按住 F9，记事本应只输入 `goodbye`，不能仍输出 `hello world`。
5. 打开外部编辑器，给 `goodbye.py` 的末尾加一个空行并保存。约一秒内表格应自动更新，不需要刷新按钮；此操作本身不得在记事本输入任何文字。随后按 F9 才允许再次加载并执行该宏。
6. 保持一个有效宏“已启用”，将该文件改为只剩 `NAME = "broken"` 并保存，或临时移出 `macros/` 目录。程序应自动显示其无效/消失状态并安全停用活动宏；随后按 F9 不得输入。测试结束后恢复原文件内容。
7. 点击 `invalid_missing_run.py` 的活动状态：它不得变为启用，F9 也不得输入。最后无论哪个宏曾启用，按 F12 禁用后按 F9 都不得输入。

## 通过与失败反馈

通过时回复：`C1 自动检测、唯一活动状态、F9 与 F12 人工验收通过`。

失败时请回复：`第 X 步失败；表格显示：…；活动状态：…；记事本文字：…；是否按 F12 启用：是/否`，并附截图（若方便）。

## 未覆盖风险

本教程验证 Windows 记事本中的真实键盘输入与保存后的自动检测；不验证游戏、管理员权限差异、第三方安全软件拦截或超长运行宏的兼容性。
