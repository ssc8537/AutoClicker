# Stage 2C-T：可编辑提示词文件与大招切人规则 Windows 验收教程

> 本轮只验收提示词文件、窗口显示和既有热键不回归。不要运行游戏输入，不要把 AI 返回的代码直接运行，也不要测试鼠标、键位配置、触发页或系统托盘。

## 准备

1. 在 `my-automation-tool` 文件夹打开 PowerShell。
2. 运行：`python main.py`。
3. 准备记事本；打开窗口后先记录窗口下方显示的“当前提示词”和“默认备份”两个绝对路径。
4. 如已有重要自定义文字，先手工复制 `ai_prompt.txt` 一份备用。

## 步骤与预期

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 在宏页点击右栏“AI 提示词”。 | 提示词正文只读；下方显示当前提示词、默认备份的两个绝对路径，并说明“用记事本修改、关闭后重新打开生效、默认备份可人工覆盖恢复”。 |
| 2 | 点击“复制提示词”，粘贴到记事本。 | 出现“提示词已复制，请粘贴给 AI”；文本含 `player.tap("R")` 后紧接 `player.sleep(1800)`，随后可直接 `player.tap("1")`；不把该规则说成动画/CD 自动检测。 |
| 3 | 关闭提示词窗口。用记事本打开“当前提示词”文件，在最开头加入一行 `我的本地测试标记` 并保存为 UTF-8。重新点击“AI 提示词”。 | 新窗口立即显示该标记，证明读取的是刚保存的文件；不需要重启程序。 |
| 4 | 关闭提示词窗口。在资源管理器把当前提示词文件改名（或移走），再重新打开窗口。 | 仍能安全显示提示词；窗口提示当前文件已从默认备份恢复。确认原路径下重新出现当前提示词文件。 |
| 5 | 可选边界检查：先备份当前提示词文件，再用记事本“另存为”非 UTF-8 编码或用其他工具写入乱码；重新打开窗口。完成后用默认备份人工覆盖当前文件恢复。 | 窗口不崩溃，提示当前文件无法按 UTF-8 读取并安全显示默认文本；程序不会自动覆盖乱码文件。完成后恢复为默认备份内容。 |
| 6 | 关闭窗口，按你已经通过的既有安全方法检查 F12 启停与 F9。 | AI 提示词不会执行宏；既有 F12/F9 行为和宏状态不变。 |

## 通过与失败反馈

通过时回复：`Stage 2C-T 提示词文件、1800ms 规则和 F9/F12 回归通过`。

失败时回复：`Stage 2C-T 第 X 步失败；当前文件路径：…；默认备份路径：…；看到的提示/文本：…；文件是否被意外覆盖：是/否；F12/F9 是否异常：是/否`，方便时附截图。

## 未覆盖风险

本教程不验证 AI 生成宏是否正确、游戏动画长度、真实游戏切人 CD、鼠标、图像识别或角色语义 API。1800ms 是作者写脚本时的保守等待规则，不是程序自动检测；真实游戏输入仍须在未来独立阶段决定和验收。
