# v021 四页 UI 外壳 - Windows 11 人工验收

> 前提：自动测试已通过。此清单只验证已实现的 UI 外壳和既有 F9/F12 行为；不要编辑 Python 宏、不要尝试启用灰色控件。

## 准备

1. 在 Windows 设置中把主显示器缩放临时设为 100%。
2. 打开 PowerShell，执行：

```powershell
cd C:\Users\s\Desktop\1-test\1-自动连点器\my-automation-tool
python main.py
```

3. 打开空白记事本，把光标放进编辑区；让程序窗口和记事本都保持可见。

## 热键与安全回归

| # | 操作 | 通过标准 | 打勾 |
|---|---|---|---|
| 1 | 启动后不按 F12，记事本有焦点时按 F9 | 没有输入，状态保持禁用 | [ ] |
| 2 | 任何页面按 F2 | 没有输入、OSD、状态变化或其他反应 | [ ] |
| 3 | 按 F12 | 显示原有绿色“全局脚本已就绪”提示，设置页状态变为启用 | [ ] |
| 4 | 鼠标移出程序窗口，在记事本按住 F9 | 持续输出小写 `hello world` | [ ] |
| 5 | 松开 F9 | 停止输出并显示原有红色结束提示 | [ ] |
| 6 | 再次按住 F9 后按 F12 | 输出立即停止，显示原有禁用提示，状态回到禁用 | [ ] |
| 7 | 关闭程序 | 不再输入、没有残留 OSD；任务管理器中没有本项目遗留的 `python.exe` 或 `pythonw.exe` | [ ] |

## 失败时的安全处理

1. 如果出现持续输入，立刻按 F12；仍未停止时关闭程序。
2. 在任务管理器结束与本项目相关的 `python.exe` 或 `pythonw.exe`。
3. 不要继续测试。记录“按了什么、看到了什么”，并附上截图和 `logs/app.log` 的相关时间。
