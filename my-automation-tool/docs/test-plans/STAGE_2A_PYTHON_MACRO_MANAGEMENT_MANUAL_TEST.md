# Stage 2A：Python 宏文件管理 Windows 小白验收教程

> 本阶段只验收新建、编辑、保存、重命名和重新加载 Python 宏。不要点击灰色的“删除”，不要测试导入导出、改快捷键、多宏、鼠标或录制。

## 启动前准备

1. 打开 PowerShell，进入 `my-automation-tool` 文件夹。
2. 运行：`python main.py`。
3. 先不要按 F12；全局热键默认禁用。
4. 本教程会创建一个名为 `stage2a_验收宏.py` 的测试文件；删除功能尚未开发，验收后它会保留在 `macros` 文件夹中。

## 操作与预期

| 步骤 | 你要做什么 | 正确结果 |
|---|---|---|
| 1 | 打开“宏库”页。 | 右侧有名称框、新建、编辑、保存、重新加载、删除；“新建”和“重新加载”可用，“删除”始终灰色。未选中有效宏时“编辑、保存”灰色。 |
| 2 | 点击“新建”。在弹出的编辑器中把名称改为 `stage2a_验收宏`，保留模板内容并点击保存。 | 返回宏页后左侧出现 `stage2a_验收宏.py`；切到“触发”页也能看到同一脚本，状态为“禁用”。 |
| 3 | 回到宏库，选中该脚本。把右侧名称改为 `stage2a_已重命名`，点击“保存”。 | 左侧文件名变为 `stage2a_已重命名.py`；触发页同步显示新文件名。旧文件名不再出现。 |
| 4 | 保持程序运行，在 `macros` 文件夹中复制 `stage2a_已重命名.py` 并将副本改名为 `stage2a_外部新增.py`；回到程序点击“重新加载”。 | 不用重启程序，宏页与触发页都会出现 `stage2a_外部新增.py`；原脚本若已启用，仍保持启用状态。 |
| 5 | 选中重命名后的脚本，点击“编辑”。在 `run(player):` 的下一行增加 `    player.sleep(10)`，点击保存；再点击“重新加载”。 | 保存成功且脚本仍是普通文字，不出现红字；重启程序前宏页和触发页也保持新名称。 |
| 6 | 再次编辑同一脚本，把函数头临时改为 `def run(other):` 并点击保存。 | 出现“保存失败”提示；关闭编辑器后重新打开该脚本，仍能看到步骤 5 保存的 `run(player)` 内容，旧文件没有被损坏。 |
| 7 | 把右侧名称改成 `bad/name` 后点击保存；再改成已有脚本的名称后点击保存。 | 两次都失败，当前脚本和已有脚本都还在，没有文件丢失。 |
| 8 | 到触发页点击该脚本的“状态”使其启用，回到宏库选中它。 | 名称框、编辑和保存灰色；必须先在触发页再次点击状态禁用，才可编辑或重命名。 |
| 9 | 在触发页启用 `hello_world.py`，准备空白记事本，按 F12 启用后按住 F9，松开后按 F12 禁用；最后再次按 F9。 | 按住期间出现 `hello world`，松开后停止；F12 禁用后再次按 F9 不再输入。Stage 2A 文件操作没有改变既有 F9/F12 安全行为。 |

## 反馈格式

全部符合时回复：`Stage 2A Python 宏文件管理和 F9/F12 回归通过`。

有问题时回复：`Stage 2A 第 X 步失败；操作前文件名：…；操作后文件名：…；看到的提示：…；触发页状态：…；F12 是否启用：是/否`。方便时附截图。

## 未覆盖风险

本阶段没有删除功能，测试文件会保留；不验证导入导出、可配置快捷键、多宏、鼠标、录制、游戏兼容性或真实鼠标输入。离屏自动测试不能替代 Windows 文件权限、中文文件名和实际编辑器显示的确认。
