# Stage 2C-U：切人时序作者规则修订 Windows 验收教程

> 本轮只验收提示词文本、复制、重开读取和既有热键不回归。不要运行游戏输入，也不要测试鼠标、键位配置、触发页、系统托盘、动画/CD 或角色检测。

## 准备

1. 在 `my-automation-tool` 文件夹打开 PowerShell。
2. 运行：`python main.py`。
3. 准备记事本；本轮无需修改提示词文件。

## 步骤与预期

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 在宏页点击右栏“AI 提示词”。 | 仍显示只读提示词与两个绝对路径；六个方向 `1→2、1→3、2→1、2→3、3→1、3→2` 都存在。普通切人是至少 `50ms`，回切是至少 `1080ms`，并说明为 `1000ms` 冷却加 `80ms` 安全余量。 |
| 2 | 点击“复制提示词”，粘贴到记事本。 | 出现“提示词已复制，请粘贴给 AI”。文本中大招示例严格相邻：`player.tap("R")`、`player.sleep(1500)`、`player.tap("1")`；示例中间没有 `50ms` 或 `1080ms` 等待，并说明结束后也可直接发 `2` 或 `3`。 |
| 3 | 关闭提示词窗口后重新打开。 | 显示的仍是本轮 50ms、1080ms、1500ms 文本，证明窗口继续从当前提示词文件读取；不需要重启程序。 |
| 4 | 关闭窗口，按你已经通过的既有安全方法检查 F12 启停与 F9。 | AI 提示词不会执行宏；既有 F12/F9 行为和宏状态不变。 |

## 通过与失败反馈

通过时回复：`Stage 2C-U 50ms、1080ms、1500ms 文本和 F9/F12 回归通过`。

失败时回复：`Stage 2C-U 第 X 步失败；看到的文本/提示：…；F12/F9 是否异常：是/否`，方便时附截图。

## 未覆盖风险

本教程不验证 AI 生成宏、游戏动画长度、真实游戏切人 CD、当前角色、鼠标、图像识别或角色语义 API。50ms、1080ms、1500ms 是作者写脚本时的保守等待规则，不是程序自动检测；真实游戏输入仍须在未来独立阶段决定和验收。
