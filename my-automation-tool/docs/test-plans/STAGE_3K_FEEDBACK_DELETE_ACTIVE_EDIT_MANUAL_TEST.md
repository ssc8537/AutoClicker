# Stage 3K 反馈修复、删除与启用宏编辑 Windows 验收教程

> 本轮验收设置页显示、宏文件操作和既有键盘安全边界。不要打开游戏；不要测试鼠标、多宏同时运行、额外热键、角色/CD/动画识别或系统托盘。

## 准备

1. 在 `my-automation-tool` 文件夹打开 PowerShell，运行 `python main.py`。
2. 准备两个有效 Python 宏；其中一个在“触发”页点击状态列，使它显示“启用”。
3. 如需检查 F9/F12，请打开记事本并将输入焦点放在空白编辑区。

## 步骤与预期

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 打开“设置”页；必要时使用鼠标滚轮或右侧滚动条。 | “共享游戏键位”下方清楚显示角色 1/2/3、战技、声骸、大招、跳跃、处决八项输入框；“保存键位设置”和提示文本位于全部输入框之后。页面不出现横向滚动条，也不再出现截图中的细线压缩。 |
| 2 | 选中已启用宏，回到“宏库”。修改右侧名称后点击“保存”；再点击“编辑”，在源码中做一个不改变接口的简单修改并保存。 | 名称/保存/编辑均可用，不再要求先禁用。改名后触发页仍只有该宏显示“启用”，名称与新文件名一致。当前已经启动的一轮不会被中途替换；下次按 F9 才读取保存内容。 |
| 3 | 选中一个未启用宏，点击“删除”，先在确认窗口点击“取消”。 | 宏文件、宏库行和触发行都保留，活动状态不变。 |
| 4 | 再次点击“删除”，确认窗口点击“确认删除”。 | 脚本从宏库和触发页消失，并进入 Windows 回收站；不是永久删除。 |
| 5 | 选中当前已启用宏，点击“删除”后点击“确认删除”。 | 该宏立即停止/停用并被移入回收站；触发页不再显示它。之后按 F9 不会启动旧宏，表现为安全失败关闭。 |
| 6 | 在记事本按既有方法确认 F12 启用/禁用和已启用宏的 F9；再按 F2。 | F12/F9 保持既有行为；F2 仍只作 UI 占位，不注册热键。没有鼠标输入，也没有多宏同时运行。 |

## 通过与失败反馈

通过时回复：`Stage 3K 键位表可见、启用宏编辑保存改名、回收站删除和 F9/F12/F2 回归通过`。

失败时回复：`Stage 3K 反馈第 X 步失败；页面/按钮状态：…；确认窗口选择：…；回收站结果：…；触发页状态：…；F9/F12/F2 是否异常：是/否`，方便时附截图。

## 未覆盖风险

本教程不验证游戏兼容性、鼠标、多个宏并行、每宏热键、角色状态、动画/CD、图像识别、自动战斗或系统托盘。删除失败时，程序会保持该宏停用并保留文件；可在宏库中重新启用或重试。
