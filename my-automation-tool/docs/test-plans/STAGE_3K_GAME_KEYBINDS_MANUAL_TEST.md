# Stage 3K：共享游戏键位与中文语义 API Windows 验收教程

> 本轮只测试键盘映射、保存和已启用宏在记事本中的键盘输出。不要打开游戏；不要测试鼠标、图像识别、自动战斗、角色/CD 判断、额外热键或系统托盘。

## 准备

1. 在 `my-automation-tool` 文件夹打开 PowerShell，运行 `python main.py`。
2. 打开记事本并把光标放在空白编辑区；记事本只用于观察键盘输出。
3. 准备一个有效 Python 宏，或在“宏库”中新建后粘贴下方内容并保存：

```python
NAME = "Stage 3K 键位测试"
HOTKEY = "f9"
MODE = "switch"
COUNT = 1
SPEED = 1.0

def run(player):
    player.切换(1)
    player.切换(2)
    player.切换(3)
    player.战技()
    player.声骸()
    player.大招()
    player.跳跃()
    player.处决()
```

## 步骤与预期

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 打开“设置”页。 | 可见“共享游戏键位”表和“保存键位设置”；八项默认值依次是 `1/2/3/E/Q/R/Space/F`。F12、F2、主题和 OSD 既有只读说明仍在。 |
| 2 | 把“角色 1”暂改为 `F9`，点击保存；再改回 `1`。 | 显示保存失败的中文说明，保留键 F9 未被写入。改回有效值后可继续。 |
| 3 | 把八项依次改为 `4/5/6/A/B/C/Space/D`，点击保存；关闭程序后重新运行并重开“设置”页。 | 显示“键位设置已保存”；重开后仍显示 `4/5/6/A/B/C/Space/D`。若将两项填为同一键，保存必须被拒绝，已保存的有效配置不变。 |
| 4 | 在“触发”页只启用上方测试宏；将焦点放回记事本，轻按 F9 一次。 | 记事本依次收到 `456abc`、一个空格、`d`（字母大小写不重要）；只运行已启用的宏。没有鼠标输入、没有游戏状态判断。 |
| 5 | 将测试宏中的第一行语义调用后加入 `player.sleep(10000)`，保存、保持记事本焦点后按 F9，再立即按 F12。 | 已开始的等待立即停止，后续动作不会继续发送；F12 的红色禁用提示仍出现。重新按 F12 恢复后，F9/F12 行为保持既有规则。 |
| 6 | 按 F2，并查看“触发”页中唯一启用状态和宏列表。 | F2 仍只作 UI 占位，不注册新热键；触发页序号、去 `.py`、唯一启用/停用、无效红字和右侧只读详情不变。 |

## 通过与失败反馈

通过时回复：`Stage 3K 键位保存、重启读取、中文语义键盘输出和 F9/F12/F2 回归通过`。

失败时回复：`Stage 3K 第 X 步失败；设置值：…；保存提示：…；重开后的值：…；记事本实际输出：…；F9/F12/F2 是否异常：是/否`，方便时附截图。

## 未覆盖风险

本教程不承诺游戏兼容性，不验证鼠标、游戏动画、角色状态、CD、图像识别、自动战斗、多宏、额外热键或系统托盘。中文语义函数只发送设置页中的物理键；它们不检测游戏状态。
