# Stage 6B · 触发设置验收

> **本轮状态列修复只需复验第 4 步。** 无需重新做其他步骤，也不打开游戏。

> **本次只在桌面或记事本测试，不打开游戏。**  
> 目标：确认每个宏都能在“触发”页直接设置，且 F12 能安全停止全部宏。

## 先准备好

1. 先关闭旧的“自动连招”。
2. 双击启动 [MyAutoPlayer.exe](../../dist/MyAutoPlayer/MyAutoPlayer.exe)。
3. 打开顶部的 **触发** 页；准备两个已有宏。没有两个宏也可以先完成步骤 1–4。

## 六步验收

| 步骤 | 你要做什么 | 看到什么才算正确 |
|---:|---|---|
| 1 | 选中一个宏，点击右侧“热键”输入框。 | 原来的字消失，提示你按键。 |
| 2 | 按一个键盘键；或在录入状态按鼠标左/右/中键、侧键。 | 表格与输入框立即更新。鼠标名称显示为 **左键 / 右键 / 中键 / 侧键1 / 侧键2**，不会显示英文内部名称。F12 不可以绑定。 |
| 3 | 点击“模式”，分别了解 **切换** 与 **按下**。 | 切换：按一次开始、再按一次停止。按下：按住开始、松开立刻停止。次数和速度修改后不必另点保存。 |
| 4 | 重新启动程序后直接打开“触发”页，**先不要点击任何宏行**。右侧详情应是灰色不可改；此时直接点击表格最右侧任意一行的状态文字。 | 被你点击的那一行第一下就立刻在 **启用** / **禁用** 间切换，不需要先选中它；坏宏只显示红色 **错误**。名称列足够宽，状态列只占很窄的位置。 |
| 5 | 把两个启用宏设成同一个热键。按 F12 使状态变绿，再按这个热键。 | 两个宏可以同时开始；这是正确行为，不会标红或互相取消。 |
| 6 | 两个宏运行时按 F12。 | 两个宏都停止。再按一次 F12 变绿后，仍可重新触发。 |

## 通过标准

六步都符合右侧“看到什么才算正确”的描述，即为通过。

## 小提示

- 右侧标题应完整显示：**触发详情（自动保存）**。
- 名称列比按键、模式、状态列更宽，长名称更容易看清。
- 所有修改会立即保存；关闭并重新打开程序后，设置仍应保留。
- 次数和速度右侧的上下箭头都应能点击；也可用滚轮调整。它们仍遵守当前项目的安全范围。

## UI 美化检查（2560×1440、133% 缩放）

打开四个顶部页面并上下查看。预期：文字、表格、按钮和底边细条都清晰完整；没有被窗口底部裁切。整体是低饱和冷灰粉、豆沙玫瑰粉细边和梅子酒红文字，不出现薄荷绿、嫩草绿或卡通蝴蝶结风格。鼠标放到按钮上有明显的浅玫瑰粉反馈；禁用按钮是灰粉色且文字仍看得清。
