# Stage 2D：触发页表格视觉修订 Windows 验收教程

> 本轮只验收触发页表格显示、既有启停入口与 F9/F12 不回归。不要运行游戏输入，也不要测试鼠标、键位配置、AI 提示词、系统托盘或其他页面功能。

## 准备

1. 在 `my-automation-tool` 文件夹打开 PowerShell。
2. 运行：`python main.py`。
3. 至少准备两个有效 Python 宏；如有一个无效宏，可额外观察红字，但不是必须条件。

## 步骤与预期

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 点击顶部“触发”页。 | 左侧表头依次为“序号、名称、按键、模式、状态”；表头和每个单元格内容均居中；没有横向滚动导致左列被遮住。 |
| 2 | 查看至少两个宏行。 | 序号从 1 开始连续；名称不显示 `.py`，但仍显示对应宏名称；按键、模式、状态内容不变。 |
| 3 | 点击一个有效宏的“状态”单元格，再点击另一个有效宏的“状态”单元格。 | 仍只有一个宏显示“启用”；先前启用项恢复“禁用”；右侧只读详情仍正确。点击非状态列不改变启用项。 |
| 4 | 如有无效宏，查看其触发行；然后按已经通过的既有安全方法检查 F12 启停与 F9。 | 无效宏的名称仍是红字且“不可启用”；序号仍连续。F12/F9 行为和宏状态不变；本轮没有运行游戏输入。 |

## 通过与失败反馈

通过时回复：`Stage 2D 触发页序号、去 .py、全列居中和 F9/F12 回归通过`。

失败时回复：`Stage 2D 第 X 步失败；看到的表头/行内容：…；点击状态后的结果：…；F12/F9 是否异常：是/否`，方便时附截图。

## 未覆盖风险

本教程不验证 AI 生成宏、真实游戏输入、鼠标、键位配置、动画/CD、角色语义 API 或系统托盘。序号和去 `.py` 仅是触发页显示，不改变磁盘文件名、宏解析或执行路径。
