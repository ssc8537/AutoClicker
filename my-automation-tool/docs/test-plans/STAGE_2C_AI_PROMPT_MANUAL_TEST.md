# Stage 2C-R：纯文本提示词与作者说明 Windows 验收教程

> Stage 2C 的原始复制功能已由用户通过。本轮只验收纯文本排版、函数说明和键位名称修复；不要测试鼠标、游戏键位配置、系统托盘、触发页改版或新热键。

## 准备

1. 在 `my-automation-tool` 文件夹打开 PowerShell。
2. 运行：`python main.py`
3. 准备一个普通文本工具（例如记事本），但不要把 AI 返回代码直接运行。

## 步骤与预期

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 保持未选中宏，点击宏页右栏“编辑”下方的“AI 提示词”。 | 出现中文窗口，文字可选择但不能修改；有“复制提示词”“关闭”。窗口中没有 Markdown 的 `#` 标题、项目符号或代码围栏。 |
| 2 | 阅读函数说明和按键参考。 | 能看懂 `player.tap`、`player.sleep`、五项元数据和 `run(player)` 的含义；按键名称是 E 战技、Q 声骸技能、R 大招，不出现“小技能”。 |
| 3 | 点击“复制提示词”，切到记事本粘贴；再选中一个有效宏并重复。 | 显示“提示词已复制，请粘贴给 AI”；未选中时是通用说明，选中后还原样包含该宏的完整源码。 |
| 4 | 关闭窗口；切到“触发”页，按原有方法验证 F12 启停和 F9 行为。 | 提示词窗口关闭；既有热键与宏状态不变。 |

## 通过与失败反馈

通过时回复：`Stage 2C-R 纯文本提示词和 F9/F12 回归通过`。

失败时回复：`Stage 2C-R 第 X 步失败；看到的符号/函数说明/键位名称：…；粘贴内容：…；F12/F9 是否异常：是/否`，方便时附截图。

## 未覆盖风险

本教程不验证 AI 生成代码的正确性、游戏兼容性、鼠标、键位映射或系统托盘；AI 返回的代码必须先由软件保存时的静态校验拦截，再由用户自行测试。
