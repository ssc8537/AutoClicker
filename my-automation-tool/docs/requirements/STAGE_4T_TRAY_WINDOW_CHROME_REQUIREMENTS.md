# Stage 4T 系统托盘与窗口控制需求规格

日期：2026-07-18
状态：实施前 READY（仅本文件范围）

## 已锁定行为

- 使用自有 Candy 粉红标题栏，右上角仅保留三个窗口动作：`×` 退出、`—` 最小化、隐藏到托盘。
- `×` 调用既有关闭路径：停止活动宏、释放输入、取消热键并退出进程；绝不隐藏到托盘。
- `—` 仅最小化到任务栏：桌面不显示窗口，但任务栏仍有窗口入口；不停止宏、不改变 F12 全局状态。
- 隐藏按钮仅隐藏主窗口：桌面和任务栏不显示窗口，托盘图标仍在；不停止宏、不改变 F12 全局状态。
- 启动时创建自有系统托盘图标（使用 Qt 标准图标，不复制案例资源）；托盘菜单仅有“显示主窗口”“隐藏主窗口”“退出程序”。单击托盘图标恢复、置前并激活主窗口。
- 维持 642 固定宽度和可垂直缩放；自有标题栏支持拖动窗口，底边支持垂直缩放。

## 安全与范围外

托盘和窗口可见性不得新建热键/启停状态机，只复用既有 F12、唯一活动宏、停止与 closeEvent 清理。若系统托盘不可用，程序仍能正常启动，隐藏按钮禁用且不隐藏窗口。

不实现开机自启、通知、托盘状态变色、主题切换、窗口位置持久化、录制、坐标鼠标、图像/游戏识别、多宏、JSON/QIM 或 AI 提示词更新。

## 验证

自动覆盖标题三动作、托盘菜单/单击恢复、不可用托盘回退、F12/关闭清理、固定宽度和垂直缩放。用户在 Windows 核对三按钮、任务栏/托盘可见性与 F9/F12 安全回归；不运行游戏输入。
