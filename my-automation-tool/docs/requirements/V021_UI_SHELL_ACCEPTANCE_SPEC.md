# v021：四页 UI 外壳验收规格（证据闭环中）

## 已锁定

- 页面：宏库、触发、功能、设置；视觉参考为 `docs/reference/quickinput-ui/` 的图片 1–4。
- 风格：采用 Quickinput Candy 粉红主题，使用 `Microsoft YaHei`；主区 `#FDF`、输入区 `#FFF5FF`、按钮/非选中标签 `#FCE`、悬停 `#FBE`、宏表头 `#FFF0FF`。图片 5 关于/更新页不采用。证据：`优秀案例1-Quickinput/Quickinput-main/source/src/init.cpp:190-329`。
- 运行边界：F9、F12、现有 OSD 必须不回归；F2 仅显示预留位置，不注册、不响应、不显示 OSD。
- 功能边界：本轮不做鼠标、窗口、录制、音效、定时、OCR、多脚本、任意热键或 Python 宏编辑写回。
- 功能页：仅视觉外壳，不产生任何运行行为。
- 窗口规则参考 Quickinput：默认 `642×510`；宽度固定 `642`；最小高度 `510`；最大高度为当前主屏高度；只允许垂直缩放；启动居中。本项目当前没有窗口几何读写实现，因此本阶段不读取、保存或恢复尺寸，不能把案例的持久化行为暗中带入。证据：案例 `MainUi.ui:5-29`、`MainUi.cpp:10-18,152-155`；本项目 `main.py:161`、`config/settings.json:3-10`。
- 页面使用顶部 `QTabWidget`，默认“宏库”（索引 0），顺序为宏库、触发、功能、设置；不保存/恢复最后打开页。原案例有五个 `128×40` 标签，本项目舍弃关于页后使用四个 `160×40` 标签填满固定窗口。证据：`MainUi.ui:281-311`、`MainUi.cpp:21-34`。
- 宏库只读展示唯一的 `hello_world.py`；允许单行选中和页面切换，但新建、编辑、保存、刷新、双击、右键、导入导出和删除均禁用且没有回调。
- 触发页只读展示 F9、`MODE`、`COUNT`、`SPEED`；允许选中唯一脚本查看固定详情，但输入控件只读，不写回 Python 文件、不改变运行配置。
- 设置页只展示既有 F12/OSD 状态与说明，不提供鼠标点击切换全局状态。
- 功能页可绘制“快速鼠标点击开关”等原版外观控件，但全部灰置并标注“后续阶段”；不注册热键、不执行点击、不修改任何设置。
- UI 只能在 `MainWindow._setup_ui()` 接入。不得新增或改变 F9/F12 注册、热键调度、播放器、OSD 生命周期、关闭清理或真实输入路径。证据：本项目 `main.py:39-216,236-250`、`src/core/hotkey_manager.py:48-174,270-282`、`src/ui/osd_window.py:14-138`。
- 自动测试继续使用现有 `unittest + PySide6` 的 offscreen `QApplication`，不引入 `pytest-qt`。验证窗口限制、默认宏库页、四页切换、只读/禁用状态、F2 未注册，以及既有 F9/F12/OSD 回归。
- 人工视觉验收在 Windows 100% 缩放、`642×510` 下对照项目内图片 1–4：检查 Candy 粉红主题、顶部四标签、控件层级和禁用“后续阶段”状态；不使用像素级截图比对。随后由用户在记事本验收 F12/F9 不回归且 F2 无反应。
