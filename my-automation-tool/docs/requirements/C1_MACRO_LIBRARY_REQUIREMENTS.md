# C1：Python 宏库管理需求规格

> 状态：自动检测与唯一活动状态已实现、待 Windows 人工验收。
>
> 本文件是 C1 的唯一规格入口。用户确认本规格后，仍须重新取得 RequirementCertifier 的实施 READY。

## 目标与边界

C1 为可信本地 Python 宏提供受控的发现、只读详情、唯一活动状态、自动文件检测和错误呈现。它不改变 Python-only、F9/F12、F2 占位、启动默认禁用、可取消执行或现有 OSD 安全边界。

不包含编辑、保存、重命名、创建、多宏并发、每宏热键、鼠标、窗口、音效、录制或定时。宏继续由用户在外部编辑器维护。

## 发现与验证

- 宏根目录固定为 `my-automation-tool/macros/`；唯一示例已从 `scripts/hello_world.py` 迁移到 `macros/hello_world.py`，不保留副本或固定回退路径。
- 只枚举根目录顶层的常规 `.py` 文件；不递归，不纳入子目录、隐藏/临时文件、`__init__.py` 或指向根目录外的链接。
- 每个候选文件必须通过现有 Python 宏约定：非空 `NAME`、`HOTKEY="f9"`、`MODE in {"switch", "down"}`、合法 `COUNT`、合法 `SPEED` 和可调用 `run(player)`。
- 优秀案例 1 已证实的可借鉴原则只有：受控宏根目录、显式格式白名单、手动 reload 与列表重建；不复制其 JSON/QIM、保存、重命名、多格式或多宏调度能力。
- 顶层 `.py`、隐藏/临时文件与链接过滤，以及 Python 宏字段校验是本项目的最小攻击面与现有运行时决定，不能表述为优秀案例已有行为。

## 状态、自动检测与失效

- 启动后没有活动宏。F9 仍保持已注册的安全热键，但处理器必须空操作：不读取空宏、不重载、不启动播放器、不发送输入，并在宏库状态区域显示“未选择有效宏”。这避免现有热键初始化读取空运行时而崩溃。
- 表格“文件校验”只表示能否加载；另设“活动状态”。点击有效行的活动状态可启用它并原子停用其他行；再次点击当前启用行则清空活动宏。无效项固定显示中文错误，不可切换。
- `goodbye.py` 是第二个有效验收宏；`invalid_missing_run.py` 是第三个故意无效、无顶层输入的验收资产。
- 使用 Qt 主线程 `QFileSystemWatcher` 监听宏根目录和已发现文件；保存、删除、改名后以 150ms 单次延迟全量重建列表并重建文件监听。不得在 F9 前隐式扫描或从监听回调执行宏。
- 已选择文件在自动同步后消失或验证失败时，必须停止播放器、`mark_finished("f9")`、清空活动选择，并显示中文原因；有效内容更新只更新详情，下一次 F9 才重新加载。
- 无效 `.py` 文件应显示文件名和中文错误原因、不可选择，并在宏库状态区域汇总提示；不以弹窗打断用户。
- F9 仍按现有机制重新加载活动文件。加载失败时必须 fail-closed：不启动播放器、不发送输入、更新中文状态，并保留既有安全 OSD。
- “默认无选择、唯一活动状态、自动监听、无效项可见不可切换”均是本项目的 fail-closed 安全决定；优秀案例未直接定义这些交互，未来实现不得把它们误记为案例事实。

## 未来实现验收

1. 空目录、有效示例、语法错误、字段错误和不可读文件均有确定列表/状态结果。
2. 默认无选择时 F9 保持安全空操作且不导致启动崩溃；启用有效宏、删除活动宏、自动检测到外部修改和无效条目均不会让 F9 误执行。
3. F9 对有效活动宏保留现有加载、取消和安全语义；F12 仍是唯一全局总开关，F2 仍仅为 UI 占位。
4. 自动测试覆盖上述情形；涉及真实输入的最终变更在记事本由用户人工验收。
