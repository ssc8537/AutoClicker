# 游戏键位与面向 AI 编程路线

> 状态：用户已确认产品方向；Stage 2B Windows 验收后，仍须由新的阶段需求门槛授权。

## 已锁定的产品决定

- 所有 Python 宏共用一个游戏键位配置；宏不保存自己的键位表，也不使用 JSON/QIM。
- 将在“设置”页以映射表提供角色切换 1/2/3、战技、声骸技能、大招、跳跃、处决、平A与闪避。
- 默认绑定为 `1/2/3/E/Q/R/Space/F/鼠标左键/鼠标右键`。F9、F12、F2 是程序保留键；映射必须拒绝保留键和重复绑定。
- 游戏键位不是第二套全局热键：只有已启用宏在 F9 触发后才会发送动作。

## 后续阶段

### Stage 3K：键盘语义（已实施，待 Windows 验收）

已实现应用自有、原子保存的 `config/game_keybinds.ini` 和设置页映射表；已发布已启用宏可调用的角色切换、战技、声骸、大招、跳跃、处决键盘语义 API。每次输入可取消，F12、异常和退出均沿用既有停止清理。此阶段不发送鼠标输入；尚待 Windows 记事本验收。

### Stage 3M：鼠标语义

在独立安全审查后实现左/右键的按下、释放、单击和有限重复；平A、闪避才会连接全局映射。必须自动验证按下—等待—释放、有限次数、停止与异常清理；用户使用记事本验收。

### 作者文档与 AI 提示词

每个 API 仅在实现、自动测试和 Windows 验收后加入 `docs/macro-authoring/`。该目录将包含入门与安全规则、已发布 API 参考、键位说明和可复制的 AI 提示词；提示词要求保留 `NAME/HOTKEY/MODE/COUNT/SPEED` 与 `run(player)`，且不得调用未发布接口。

## 明确不做

不复制优秀案例的源码、资源、品牌、图像识别、战斗判断或角色 AI；不增加多宏并发、额外全局触发键、JSON/QIM 宏、无限鼠标循环、录制或游戏兼容性承诺。
