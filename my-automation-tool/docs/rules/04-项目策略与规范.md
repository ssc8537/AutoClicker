## 十二、文件拆分防幻觉策略

### 为什么需要拆分文件
项目文档和测试文件如果写得太长，AI 在阅读时容易产生幻觉（遗漏内容、混淆信息）。
把大文件拆成小文件，每个文件只包含少量内容，AI 能更准确地理解和执行。

### 拆分原则
1. **每个文件不超过 100 行或 3 个阶段内容**
2. **测试计划拆分到 docs/test-plans/ 目录**，按阶段分组
3. **所有测试文件通过根目录的 STAGE_TEST_PLAN.md（总索引）引用**
4. **新增阶段时不修改旧文件**，追加到 STAGE_5+.md 末尾或新建文件

### 接手 AI 如何阅读
1. 先读 STAGE_TEST_PLAN.md（总索引），了解整体结构
2. 根据当前阶段打开对应的测试文件
3. 如果当前阶段是进行中的，看该阶段的测试清单
4. 如果当前阶段还没实现，看需求说明即可

### 文件索引更新规范
- 新增阶段文件后，必须更新 STAGE_TEST_PLAN.md 总索引的表格
- 修改文件路径或文件名后，必须更新所有引用该文件的文档

## 十三、编码规范（防乱码）

### 此规则的重要性
本项目所有文档使用中文编写。如果文件编码出错，中文将变成乱码（如"?????"），
导致 AI 无法阅读、文档体系崩溃。此规则为最高优先级的防错规范。

### 核心规则
**写任何包含中文的文件时，必须使用 apply_patch 工具，不允许使用 PowerShell 的命令行传中文。**

### 原因
PowerShell 默认使用 GBK 编码（非 UTF-8），通过以下方式传中文会乱码：
- PowerShell 的 `@文字@` heredoc 传中文
- `python -c "包含中文的字符串"`
- PowerShell 的 `-Command` 参数直接传中文

`apply_patch` 工具能正确处理 UTF-8 编码，不会出现乱码问题。

### 正确的做法
1. 新建中文文件：用 `apply_patch` 的 Add File 模式
2. 修改中文文件：用 `apply_patch` 的 Update File 模式
3. 运行 Python 脚本处理文件：先用 `apply_patch` 创建 .py 文件，再执行它

### 错误的做法（禁止）
- 用 PowerShell 的 `Set-Content` 写中文文件
- 用 PowerShell 的 `@文字@` heredoc 传中文给 Python
- 用 `python -c "中文"` 处理包含中文的文件

### 如果出现乱码怎么办
1. 直接用 `apply_patch` 的 Add File 重写该文件（不要尝试修修补补）
2. 或者先用 Python 创建 .py 脚本文件（用 apply_patch），再执行它创建 .md 文件
