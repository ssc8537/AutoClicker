# 交接协议与日志规范

> 本文档是项目信心需求文档的子文件（二）。
> 对应原 项目信心需求文档.md 的三、四、五 章节。

---

## 三、交接协议（每个 AI 结束时必须完成）

### 必须更新的文件
| 文件 | 更新内容 |
|------|---------|
| PROJECT_OUTLINE.md | 当前进度表 + 最新版本号 |
| REVIEW_LOG.md | 追加本次审查记录（含日志） |
| PROJECT_TASKS.md | 完成项标记 [x] |
| 项目管理/LATEST.txt | 指向新版本号 |

### 每轮审查记录必须包含
1. 完成了哪些子任务
2. 修改了哪些文件（路径 + 改动说明）
3. 自动化测试的运行结果（关键日志行）
4. 发现了什么 Bug、怎么修的
5. 下一步要做什么（给下一个 AI 的明确指令）

### 版本快照
- 复制 PROJECT_OUTLINE.md 到 项目管理/vXXX/
- 复制 REVIEW_LOG.md 到 项目管理/vXXX/
- 更新 项目管理/LATEST.txt = vXXX

---

## 四、日志规范（日志 = AI 的眼睛）

- 所有程序运行日志写入 logs/app.log（RotatingFileHandler）
- 日志格式：时间 [级别] 模块名 信息
- 关键事件必须打日志：程序启动/退出、热键注册/注销、热键触发/抑制、状态切换、异常错误
- 自动化测试脚本必须输出可读的结论到终端
- 关键测试结果必须复制到 REVIEW_LOG.md 中存档

---

## 五、进度控制

用户通过以下方式控制项目进度：
1. 打开 PROJECT_OUTLINE.md -> 看到进度表
2. 打开 REVIEW_LOG.md -> 看到每个 AI 做了什么、测试结果
3. 按 STAGE_TEST_PLAN.md 的阶段测试计划手动测试
4. 告诉 AI：测试通过了继续下一步，或这里不对劲

如果换了另一个 AI 接手：
1. 先读 项目信心需求文档.md
2. 按 Step 1 的顺序读所有文档
3. 查看 LATEST.txt 确认最新版本
4. 查看 vXXX/ 下的历史版本了解完整决策链
