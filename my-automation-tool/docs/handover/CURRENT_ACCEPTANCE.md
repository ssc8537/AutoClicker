# 按键记录与30分钟保存修复：Windows 验收

> 验收状态：2026-07-23 用户确认以下教程全部通过。

> 安全准备：先保存正在编辑的宏。验收只使用普通视频或桌面，不要在重要操作中测试宏按键。

## 验收表

| 序号 | 验收项目 | 操作 | 正确结果 |
|---:|---|---|---|
| 1 | 临时输入不弹状态 | 打开“开发连招 → 打开按键记录窗口”，点击右下角“可以在此输入按键”，连续输入几个宏触发键 | 文字正常进入临时框；按键窗最近事件不跳动；宏不启动；没有宏启动/停止 OSD。若正在录像，原始按键日志仍会记录物理边沿。 |
| 2 | 最小化后重新出现 | 鼠标移入按键窗，点击“—”最小化；回到开发连招页，再点“打开按键记录窗口” | 原来的按键窗立即恢复，位置和大小保持，不需要重启程序。 |
| 3 | 桌面声音可调 | 在“桌面声音音量”把滑块依次调到100%、150%、200%，同时播放一段音乐观察预检条；再录一小段并播放 | 百分比实时显示；预检条随音量变化；成片音轨1明显变大。声音失真时调低，推荐先用150%。 |
| 4 | 时钟放大 | 查看按键窗右上角`HH:mm:ss:fff`时间，再把窗口缩放到不同大小 | 时间字号约为详情文字2倍，随窗口同比缩放，不遮挡关闭/最小化按钮。 |
| 5 | 保存不再报206 | 日常可先录制1–2分钟并点“保留过往”确认能正常生成；需要时再进行真实30分钟测试 | 保存成功生成`raw.mp4`和字幕/日志，不再出现“文件名或者扩展名太长 / Windows error 206”。自动测试已模拟180组视频、桌面音频和麦克风路径。 |

## 通过标准

五项均符合“正确结果”即可通过。真实30分钟连续录制不属于本轮强制测试；如果你以后实际录满30分钟，再补充确认第5项即可。
