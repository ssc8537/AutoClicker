# MyAutoPlayer 总阶段路线（唯一规划入口）

本文件替代早期 JSON、多热键和旧 UI 任务表。它只列出已确认能力与必须先由用户打开范围的后续能力；不得因为路线中有候选就直接写代码。

## 已完成

1. **安全单宏基础**：Python `run(player)`、F9/F12、停止/异常/退出释放、OSD、唯一活动宏。
2. **宏库与作者体验**：发现、静态校验、新建、编辑、原子保存、改名、重载和回收站删除。
3. **游戏动作**：可配置键盘语义与受限左右鼠标 API（1–100 次、最少 10ms）。
4. **桌面体验**：Candy 粉色窗口、任务栏最小化、托盘隐藏/恢复、安全退出。
5. **V020 备份**：已发布 GitHub 标签 `v020-stage4t-and-ui-polish`。
6. **Windows 便携版**：用户已完成 EXE、图标、持久化和热键 Windows 验收。

## 当前小阶段：Stage 6 多宏独立触发（设计中）

目标是在不允许并行输入的前提下，让每个已保存 Python 宏拥有不同的安全触发键。开始实现前必须锁定可用键范围、重复/冲突回退、F12 停止和触发页交互；由案例专家完成只读取证后再写代码。

## 后续候选（必须一次只选一个）

1. **多宏与触发设计**：先锁定是否允许并行、每个宏的触发方式、冲突和 F12 停止语义；未锁定前不写代码。
2. **已验收 API 的 AI 作者提示词更新**：仅在用户明确要求后，把已验收的键盘/鼠标 API 写入提示词和作者手册。
3. **发布形态**：若用户需要，再单独设计 Windows 可执行包/安装方式；当前 GitHub 源码发布已完成。

## 完成定义

“完整程序”不是无限加入案例功能：它等于用户选择的候选全部逐项完成自动测试与 Windows 验收。坐标/图像识别、录制、游戏判断、JSON/QIM 等仍不在路线内。
